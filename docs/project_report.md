# 软件度量自动化工具项目设计与实现报告

## 1. 项目概述

本项目实现了一个面向教学与课程实践的中小型软件度量自动化工具 `metrics_tool.py`，用于将软件度量课程（尤其是面向对象度量）中的核心模型应用到真实项目。工具覆盖需求分析、设计、编码和评估阶段的关键度量点，并输出结构化 JSON 报告，便于后续预测、评估和改进。

**项目创意点**：将“设计图输入 + 源码静态分析 + 项目管理预测（工作量/成本/周期）”整合到同一工具链，以课程知识驱动“可落地”的工程决策。

---

## 2. 需求分析

### 2.1 功能需求

1. **输入管理**：读取 Java 源代码目录、类图/用例图/流程图 JSON、人员与成本参数。
2. **代码度量**：统计 LoC、近似圈复杂度。
3. **面向对象度量（CK）**：WMC、DIT、NOC、CBO、RFC、LCOM。
4. **面向对象度量（LK）**：NOM、NOA、NOPM、MPC、DAC（教学近似版本）。
5. **项目级预测**：基于 COCOMO Basic 估算工作量、开发时间、人员配置下周期、成本。
6. **报告输出**：生成项目级与类级 JSON 报告。

### 2.2 数据需求

- **源代码实体**：类、方法、字段、继承关系、方法调用。
- **设计输入实体**：`class_diagrams`、`use_cases`、`flow_charts`。
- **管理输入实体**：`persons`（人员）、`hourly_rate`（时薪）。

### 2.3 非功能需求

- **可扩展性**：度量点可继续扩充（如功能点、特征点）。
- **易用性**：命令行单命令执行；JSON 输入输出便于集成。
- **可维护性**：核心算法函数化，具备单元测试。

---

## 3. 系统架构设计

采用四层架构：

1. **输入层**：解析代码目录与设计 JSON。
2. **分析层**：Java 代码轻量解析（正则 + 语法片段抽取）。
3. **度量层**：CK/LK/复杂度/规模/估算算法计算。
4. **输出层**：结构化报告（项目摘要 + 类级明细）。

### 3.1 关键模块

- `parse_java_file`：抽取单类基础信息。
- `compute_hierarchy_metrics`：补齐 DIT/NOC。
- `estimate_effort_cost`：COCOMO Basic 估算。
- `aggregate`：汇总项目维度指标。
- `build_report`：端到端分析入口。

---

## 4. 度量模型与算法说明

## 4.1 CK 模型（实现）

- **WMC**：类复杂度总和（本工具用类级近似圈复杂度表示）。
- **DIT**：继承深度。
- **NOC**：子类数量。
- **CBO**：类耦合（外部对象调用数量近似）。
- **RFC**：响应集合（本类方法数 + 外部调用方法种类）。
- **LCOM**：方法间共享字段稀疏度（基于字段引用集合近似）。

## 4.2 LK 模型（实现）

- **NOM**：方法数量。
- **NOA**：属性数量。
- **NOPM**：公有方法数量。
- **MPC**：消息传递耦合（方法调用次数）。
- **DAC**：数据抽象耦合（抽象数据类型引用数近似）。

## 4.3 传统与项目管理度量

- **LoC**：非空非注释代码行。
- **圈复杂度（近似）**：`1 + 分支/逻辑关键词计数`。
- **工作量/成本/时间**：COCOMO Basic（Organic）+ 人员参数修正。

## 5. 自动化实现结果展示

示例命令：

```bash
python3 src/metrics_tool.py \
  --source /path/to/java/project \
  --design src/sample_design.json \
  --persons 5 \
  --hourly-rate 150 \
  --output metrics_report.json
```

输出包含：

- `project`：项目平均 CK/LK 指标、LoC、类数量。
- `estimation`：`effort_person_month`、`schedule_month`、`cost` 等。
- `design_input`：类图/用例/流程图数量。
- `classes`：每个类的详细度量点。

## 6. 预测、评估与改进分析

### 6.1 与实际应用比较方式

可将工具输出与真实项目数据对比：

1. **进度偏差**：估算 `schedule_month` 与实际迭代周期对比。
2. **成本偏差**：估算 `cost` 与财务工时统计对比。
3. **质量信号**：高 LCOM/高 CBO/高 WMC 类通常对应缺陷热点。

### 6.2 工具准确性与有效性

- **准确性**：本版本是教学近似实现，利于理解指标内涵；生产级可接入 ASTParser（如 Eclipse JDT）提升精度。
- **实用性**：低接入成本，适合课程实验与项目初筛。
- **有效性**：可快速定位“高复杂-高耦合-高风险”模块，支持重构与资源配置决策。

### 6.3 改进方向

1. 接入 Eclipse ASTParser / JavaParser 获取语义级解析。
2. 增加功能点、用例点、特征点估算模型。
3. 引入缺陷数据训练风险预测模型（度量 + ML）。
4. 提供 Web 可视化仪表盘与趋势分析。

---

## 7. 结论

本工具覆盖软件度量课程大部分核心知识点，并将代码度量、OO 度量、项目管理度量融为一体，既满足教学要求，也可用于真实项目的早期评估与持续改进。
